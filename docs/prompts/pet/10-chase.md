# 10 · 帧动画 · chase（追逐鼠标）

> **对应文件**：`src/assets/sprites/chase-0.png` … `chase-3.png`
> **帧数**：4
> **单帧尺寸**：128 × 128 PNG RGBA，透明背景
> **循环**：是
> **帧率**：8 fps（比普通 running 快，表达兴奋）

## 触发场景（代码侧）

`useChaseCursor.ts`：用户在 pet 窗口上抖动鼠标时，pet 开始追赶鼠标光标，直到抵达或超时。
当前用的是 `running` sprite，独立 `chase` sprite 让它在追光标时的表情更有冲劲。

## 动作描述

柯基**向右侧冲刺**（侧面朝右），比 `running` 更**兴奋、更低伏**——像狗发现球后冲过去。

- 第 0 帧：身体低伏（比 running 更贴地 2-3 像素），前腿蹬出
- 第 1 帧：四脚离地（腾空），耳朵被风向后吹
- 第 2 帧：前腿落地、后腿蹬出
- 第 3 帧：四脚离地第二次，为下一循环做准备

核心特征：
- **舌头吐出**（粉色，占嘴右侧 3-4 像素），贯穿全程
- **眼睛睁大**（有神、聚焦），瞳孔方向朝前
- **耳朵向后吹**（被风压贴头）
- **尾巴绷直朝后**（不是摇摆，是"对准目标线"的硬朗尾巴）

## 注意

- 视角侧面朝右，CSS 会按运动方向左右翻转，因此画右向即可。
- 这是循环动画，第 0 和第 3 帧衔接要顺。
- 和 `running` 的区别在**情绪强度**：running 是日常小跑，chase 是"专注追猎"。
