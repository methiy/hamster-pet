# 17 · 帧动画 · scratch（挠痒）

> **对应文件**：`src/assets/sprites/scratch-0.png` … `scratch-3.png`
> **帧数**：4
> **单帧尺寸**：128 × 128 PNG RGBA，透明背景
> **循环**：是
> **帧率**：6 fps

## 触发场景（代码侧）

idle 状态下的随机动作变体，偶发（比如每 2-3 分钟出现一次）。给桌宠增加"生动感"，让它不是永远保持 idle 呼吸。

## 动作描述

柯基**坐姿正面**，**抬起一只后腿**挠自己耳朵/脖子侧，像狗挠痒的动作。

- 第 0 帧：坐姿，一只后腿（比如右后腿）开始**抬起**，指向颈侧
- 第 1 帧：后腿**触到耳后**，开始**快速来回蹬**
- 第 2 帧：后腿蹬动的另一端（蹬向外）
- 第 3 帧：后腿收回，为下一循环做准备

核心特征：
- **坐姿**（不是站姿）：后半身坐地、前两腿撑地
- **抬起的那条后腿**是核心视觉：脚掌接近耳朵/脖侧
- **头部稍微侧歪**（跟着被挠的方向）
- **舌头可以轻轻吐出**（1-2 像素）"爽"的感觉
- 眼睛半眯
- 尾巴可以随挠动幅度小摆

## 注意

- 循环动画：帧 1-2-3 就是在"蹬蹬蹬"的不断挠。
- 抬起的后腿建议用右腿（画布里是观众视角的左侧），因为 CSS 翻转时逻辑一致。
- 不要画得很激烈，要可爱松弛。
