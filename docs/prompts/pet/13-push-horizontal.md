# 13 · 帧动画 · push-horizontal（水平方向推拉窗口）

> **对应文件**：`src/assets/sprites/push-h-0.png` … `push-h-3.png`
> **帧数**：4
> **单帧尺寸**：128 × 128 PNG RGBA，透明背景
> **循环**：是
> **帧率**：6 fps

## 触发场景（代码侧）

`usePushAnimation.ts` 里当检测到用户沉迷视频/游戏时，pet 跑去**推其他应用窗口**。push 方向有 left/right/up/down 四种；**水平方向（left/right）共用这组 sprite**，代码在 `left` 时用 CSS 做**水平翻转**。所以本文件画**向右推**即可。

## 动作描述

柯基**侧面朝右**，身体**前倾**用前爪**抵住**右侧一个看不见的平面（就是那个要推的窗口），整个身子向右用力。

- 第 0 帧：蓄力姿态（前腿伸直抵右、后腿蹬地、身体呈斜线）
- 第 1 帧：发力瞬间（前腿更用力、脸部表情紧绷）
- 第 2 帧：身体微向左反弹（发力回荡），仍保持抵着的姿态
- 第 3 帧：回到接近第 0 帧的蓄力状态

核心特征：
- **前腿伸直，直指右侧画布边缘**（前爪要贴到画布接近右边 8-12 像素的位置）
- **身体整体斜线倾斜**约 15°，从左下到右上
- **脸部表情用力**：嘴咬紧、眼睛眯起、眉毛压低
- **后腿蹬直**，支撑身体前推的力量
- 尾巴平伸或略向下（被用力带着）

## 注意

- 循环动画，两帧一组交替（0-1-2-3 → 0）模拟"使劲推→微微回弹→再推"的节奏。
- 视角**侧面朝右**；代码中 dir='left' 时 CSS 会用 `transform: scaleX(-1)` 翻转显示。
- 不要画那个被推的窗口/物体——只画柯基本身。
